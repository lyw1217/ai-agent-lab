# Refactoring Workflow

## 언제 쓰면 좋은가

- 동작은 유지하면서 구조를 개선할 때
- 중복 제거, 함수 분리, 타입 정리를 할 때
- 큰 변경 전에 안전한 작은 단계로 나눌 때

## 기본 흐름

1. 현재 동작을 설명하게 한다.
2. 리팩터링 목표와 변경하지 않을 동작을 명시한다.
3. 작은 단위로 변경하게 한다.
4. 테스트 또는 snapshot으로 동작 보존을 확인한다.
5. 사람이 API 변화와 edge case를 검토한다.

## 기록할 것

- 리팩터링 전후 diff 크기
- 테스트 통과 여부
- 가독성 개선 여부
- 오히려 복잡해진 부분
