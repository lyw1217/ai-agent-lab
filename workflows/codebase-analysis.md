# Codebase Analysis Workflow

## 언제 쓰면 좋은가

- 처음 보는 repo 구조를 빠르게 파악할 때
- 리팩터링이나 기능 추가 전에 영향 범위를 정리할 때
- onboarding 문서나 architecture map을 만들 때

## 기본 흐름

1. repo 구조와 주요 entrypoint를 찾는다.
2. 빌드/테스트/실행 명령을 확인한다.
3. 핵심 모듈과 데이터 흐름을 요약한다.
4. 변경 대상과 위험 영역을 분리한다.
5. 사람이 검토할 질문 목록을 남긴다.

## 기록할 것

- 사용 도구
- 사용한 프롬프트
- 분석 결과의 정확도
- 사람이 보완한 내용
