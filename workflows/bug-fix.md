# Bug Fix Workflow

## 언제 쓰면 좋은가

- 재현 가능한 버그를 빠르게 분석하고 수정할 때
- failing test나 에러 로그가 있을 때
- 작은 범위의 회귀를 고칠 때

## 기본 흐름

1. 증상, 재현 방법, 기대 동작을 정리한다.
2. 관련 파일과 최근 변경을 찾는다.
3. 가능한 원인을 1~3개로 좁힌다.
4. 가장 작은 수정으로 고친다.
5. 테스트를 추가하거나 기존 테스트로 검증한다.

## 주의할 점

- 에러 로그만 보고 과도하게 추측하지 않는다.
- 관련 없는 리팩터링을 함께 넣지 않는다.
- AI가 수정한 코드는 사람이 diff를 읽고 확인한다.
