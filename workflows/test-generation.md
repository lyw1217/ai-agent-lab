# Test Generation Workflow

## 언제 쓰면 좋은가

- 기존 코드에 테스트가 부족할 때
- 버그 수정 후 회귀 테스트를 추가할 때
- edge case를 빠르게 넓혀보고 싶을 때

## 기본 흐름

1. 테스트 대상 함수/컴포넌트의 계약을 정리한다.
2. 기존 테스트 스타일과 fixture를 먼저 읽게 한다.
3. 정상/오류/경계 케이스를 목록화한다.
4. 테스트를 생성하고 실제로 실행한다.
5. 실패 원인이 테스트 문제인지 코드 문제인지 분리한다.

## 주의할 점

- 구현 세부사항에 과하게 의존한 테스트를 피한다.
- 의미 없는 snapshot 남발을 피한다.
- flaky test 가능성을 사람이 확인한다.
