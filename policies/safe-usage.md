# Safe Usage Policy

## 민감 정보

- 회사 기밀, 고객 데이터, 인증 토큰, 개인 정보는 AI 도구에 입력하지 않는다.
- 실험에는 개인 토이 프로젝트, 오픈소스 repo, 샘플 데이터를 사용한다.
- 실제 업무 코드와 유사한 예제가 필요하면 민감 정보를 제거한 재현 repo를 만든다.

## 권한과 실행

- agent에게 destructive command를 맡기기 전에 사람이 확인한다.
- 외부 서비스 연동 시 권한 범위를 최소화한다.
- 자동 실행 hook은 테스트/린트처럼 복구 가능한 작업부터 시작한다.

## 기록

- 사용한 prompt, command, 설정은 가능한 범위에서 기록한다.
- 실패 사례와 위험 사례도 숨기지 않고 남긴다.
